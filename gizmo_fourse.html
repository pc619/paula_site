Gizmo - Physical Computing | University Project

Summary

Brief

To create a cardboard machine with actuators and sensors to a create a meaningful and iterative physical interaction.

To achieve the brief above, I built a robotic hand that types letter into a laptop when the user inputs morse code into it (through a touch resistor). The user inputs morse code and when they are done with a word, they say 'go' into a microphone so that the robot starts to type.

While the robot is typing an LED will turn red to indicate that the user cannot input more morse code. As soon as it is done typing the word, the LED will turn red and the user can continue to input morse code!

The Process

The process undertaken to complete this project required a significant amount of prototyping and testing to get the machine to work as expected. Although the process became frustrating (mostly due to the unreliability of cardboard for such a machine), this project has taught me a lot about robotics.

Solidifying the concept

After some brainstorming potential ideas, I decided that the aim of my project would be to make robot that translates inputted morse code into letters and types it into the keyboard. The interactive elements that the robot will contain are the following:

01

Touch Sensor

The user will tap in the morse code into the touch sensor, it is easier and more seamless to use than a mormal switch.

02

Sound sensor

When the user is done inputting a word, they can say ‘go’ into the microphone to signal the robot that it can start typing the word.

03

RGB Led

An RGB led will turn red when the robot is typing and green when the user can start to type the next word.

Designing extension mechanism

The extension mechansim needed to pass two criteria:

01

The hand needs to always be kept horizontal.

02

The hand should always remain at the same height.

I came up with two geometrical mechanisms and coded them on Matlab. In the end, I went for the 'crane' design because it was Less mathematically complicated, hence more opportunities to explore mechanisms. The disadvantage of the 'crane' design is that it looks less like an arm than the other one.

Designing and testing the hand

The hand mechansim needed to pass two criteria:

01

Must look similar to real fingers.

02

One of the fingers must have enough force to push down a key on the computer.

Originally, the mechanism for the finger was going to work by a string that pulled the finger - therefore the finger needed to have joints. However, this mechanism didn't have enough force to push the key. Therefore, the final mechanism was changed to a simple rotation mechanism.

String Mechanism

Motor pulling string to bend and press (seen in picture 1). When motor is released, finger returns to original position thanks to an elasstic band.

Rotation Mechanism

The motor is directly attached to the finger. When the finger is up (not pressing) it is at an angle of 100º. When the finger is pressed, it lowers to an angle of 35º for 200ms.

Building mechanism and testing

I first made a lo-fi prototype to ensure the basics of the mechanism worked, then I made a more stable model and added the motors to test it.

After testing the above mechanism using the Elegoo Mega Board, there were a few problems that needed to be resolved:

  • The two SG90 servos did not have enough strength to fold the mechanism up again once it had been extented.
  • After usage, the holes on the cardboard where the axis of rotation were inserted grew too much, resulting in an unstable structure due to high clearance.
  • The hand was too heavy and did not stay horizontal (slanted downward) and thus fell on top of the keyboard.

Therefore, a new mechanism was made with the following changes:

  • Bought new servos (MG995 model) which can move up to 15kg.
  • Put wooden depressors on the cardboard pieces to reduce the clearance in rotation.
  • To make the hand more stable, the mechanism was purely symmetrical and I attached a rubberband between the hand and the arm.

Building the platform & cabling

I first made a lo-fi prototype to ensure the basics of the mechanism worked, then I made a more stable model and added the motors to test it.

Platform

The platform had the following features:

  • An MG995 servo embedded into two honeycomb cardboard pieces.
  • A circular 5 ply corrugated piece of cardboard reinfored with four wooden tongue depressors stuck to the blades of the servo.
  • Four metal saddle clamps stuck below the platform to provide support and to allow the platform to slide (metal ensures there is less friction).

Attachment

The extension & hand mechanisms were attached to the rotating platform in the following manner:

  • The base was drilled into the platform and held still with a small metal plate.
  • To counteract the weight of the hand, the base was placed behind the center of the platform & 400g of screws were arranged onto the back of the base.
